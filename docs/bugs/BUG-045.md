# BUG-045 — `/profile` top-bar "More" overflow has no action

**Status:** Fixed
**Severity:** Low
**Opened:** 2026-05-17
**Source:** [button-link-audit-2026-05-17.md §Profile](../button-link-audit-2026-05-17.md)
**Area:** profile

## Summary

The top-bar **More** overflow button on `/profile` opens no menu, route,
dialog, or API call.

## Steps to reproduce

1. Sign in and open <https://gentle-rock-070070c0f.7.azurestaticapps.net/profile>.
2. Click the **More** (overflow) icon button.
3. Observe: nothing happens.

## Expected (decision required)

Two reasonable options:

- Open a CDK action sheet with profile / account actions (Sign out,
  Delete account, About, etc.).
- Remove the button — the Account-section **Sign out** is already the
  discoverable primary affordance and `D4` of `docs/sign-out-plan.md`
  explicitly defers a second top-bar action.

## Actual

Inert button.

## Likely source

- `frontend/projects/saturdaze/src/app/pages/profile/profile.page.html`

## Acceptance

- Either the button is removed, or it opens a menu with at least one
  functional action.
- E2E spec asserts the chosen behaviour.
