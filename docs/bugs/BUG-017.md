# BUG-017 — Signup "Terms" link uses `href="#"`

**Status:** Fixed
**Severity:** Low
**Opened:** 2026-05-17
**Source:** [button-link-audit-2026-05-17.md §Signup](../button-link-audit-2026-05-17.md)
**Area:** legal / nav

## Summary

The **Terms** link in the legal-acceptance copy beneath the signup form is
rendered with `href="#"`. On the deployed app it navigates away from
`/signup` rather than opening a terms document.

## Steps to reproduce

1. Open <https://gentle-rock-070070c0f.7.azurestaticapps.net/signup>.
2. Click the inline **Terms** link.
3. Observe: URL becomes `/signup#`, scroll snaps to top.

## Expected

The Terms link opens the actual Terms of Service document — either via a
real internal route (`/terms`), an external policy URL, or a static page.

## Actual

`href="#"` traps the user on `/signup` with a scroll jump.

## Likely source

- `frontend/projects/saturdaze/src/app/pages/signup/signup.page.html`

## Acceptance

- The link either:
  - routes to a real `/terms` page rendered from canned content,
  - links to an external policy URL with `target="_blank" rel="noopener"`,
  - or is replaced with plain text until the document exists.
- Playwright spec asserts the chosen behaviour.

## Notes

Pair with [BUG-018](BUG-018.md) — Privacy Policy has the same defect.
