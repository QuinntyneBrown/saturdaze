# BUG-027 — `/weekend` detail-pane "Regenerate" has no action

**Status:** Fixed
**Severity:** Medium
**Opened:** 2026-05-17
**Source:** [button-link-audit-2026-05-17.md §Home](../button-link-audit-2026-05-17.md)
**Area:** weekend / home

## Summary

The **Regenerate** button inside the detail pane of `/weekend` produces
no visible change. Unlike the quick-action card ([BUG-023](BUG-023.md))
this button does not even use `href="#"` — it's simply not wired.

## Steps to reproduce

1. Sign in and open <https://gentle-rock-070070c0f.7.azurestaticapps.net/weekend>.
2. Expand the detail pane for a day.
3. Click **Regenerate**.
4. Observe: no route, no dialog, no API call.

## Expected

Click opens a confirm dialog scoped to *this day only* ("Regenerate
Saturday?") and on confirm calls a per-day regenerate endpoint
(`POST /api/weekends/current/days/saturday/regenerate` or equivalent).
The detail pane refreshes with the new plan; locked blocks stay put.

## Actual

Inert button.
