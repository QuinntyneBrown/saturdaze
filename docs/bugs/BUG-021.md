# BUG-021 — `/weekend` toolbar "Share with Sara" button has no action

**Status:** Fixed
**Severity:** Medium
**Opened:** 2026-05-17
**Source:** [button-link-audit-2026-05-17.md §Home](../button-link-audit-2026-05-17.md)
**Area:** weekend / home

## Summary

The toolbar **Share with Sara** button on `/weekend` produces no visible
state change. Share functionality (a read-only family-share link) is
absent from the app entirely.

## Steps to reproduce

1. Sign in and open <https://gentle-rock-070070c0f.7.azurestaticapps.net/weekend>.
2. Click the toolbar **Share with Sara** button.
3. Observe: nothing happens.

## Expected

Click opens a share CDK dialog that:

- Creates (or fetches) a read-only family-share link for the current
  weekend plan.
- Renders the link with a one-tap copy affordance.
- Offers a native `navigator.share()` invocation on mobile.

## Actual

Inert button.

## Likely source

- `frontend/projects/saturdaze/src/app/pages/home/home.page.html`
- `frontend/projects/saturdaze/src/app/pages/home/home.page.ts`
- (Backend) `Saturdaze.Api` — a `POST /api/weekends/{id}/share` endpoint
  and `WeekendShareToken` persistence model are needed.

## Acceptance

- Click opens a share dialog (CDK, follows the dialog conventions).
- The dialog displays a real share URL pointing at a public read-only
  weekend page (or returns an existing token if one is active).
- Copy-to-clipboard works.
- E2E spec covers the dialog open + link copy.

## Notes

The same share dialog should be reused by the quick-action and detail-pane
share CTAs ([BUG-025](BUG-025.md) and similar).
