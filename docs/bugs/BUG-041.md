# BUG-041 — `/saved` "Remix" has no action

**Status:** Fixed
**Severity:** Medium
**Opened:** 2026-05-17
**Source:** [button-link-audit-2026-05-17.md §Saved](../button-link-audit-2026-05-17.md)
**Area:** saved

## Summary

The **Remix** button on each saved-weekend card produces no visible state
change. Creating a new draft seeded by a saved weekend (with variation)
is a major saved-weekend product feature and is unimplemented.

## Steps to reproduce

1. Sign in and open <https://gentle-rock-070070c0f.7.azurestaticapps.net/saved>.
2. Click **Remix** on any saved card.
3. Observe: no route, no dialog, no API call.

## Expected

Click creates a new draft weekend plan seeded from the saved weekend
with variation (different activities pulled from likes / dislikes, same
commitments). On success, route to `/weekend` (or `/itinerary`) showing
the new draft.

## Actual

Inert button.

- Click creates a new draft via API.
