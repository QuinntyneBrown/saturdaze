# BUG-046 тАФ `/components` and `/dialogs` gallery routes are publicly accessible with non-functional demo buttons

**Status:** Fixed
**Severity:** Low
**Opened:** 2026-05-17
**Source:** [button-link-audit-2026-05-17.md ┬зDemo/Gallery Pages](../button-link-audit-2026-05-17.md)
**Area:** gallery / hygiene

## Summary

The `/components` and `/dialogs` design-gallery routes are reachable on
the deployed production URL. They render many `<sd-button>` and
`<sd-icon-button>` instances that intentionally do nothing тАФ they are
review surfaces for components and dialog shells, not product features.
They are nevertheless public, indexable, and clutter the audit signal
because every demo button looks broken.

## Steps to reproduce

1. Open <https://gentle-rock-070070c0f.7.azurestaticapps.net/components>
   (no sign-in required).
2. Click any visible button тАФ none have product effects.
3. Same for `/dialogs`.

## Expected (decision required)

One of:

- Keep the routes public but gate them behind `data-demo` markers and a
  Playwright filter so the production-control audit can skip them
  cleanly.
- Hide the routes from production by gating them behind a feature flag
  (`environment.galleryRoutes`) or a local-only flag.
- Move them into a separate `gallery` sub-application that isn't deployed
  to the public SWA.

## Actual

Routes are public and audit-noisy.

## Likely source

- `frontend/projects/saturdaze/src/app/app.routes.ts`
  (gallery route registration)
- `frontend/projects/saturdaze/src/app/pages/components-gallery/`
- `frontend/projects/saturdaze/src/app/pages/dialogs/`
- `frontend/projects/saturdaze/src/environments/environment*.ts`

## Acceptance

- Product owner picks one of the three options.
- Either:
  - Routes are removed from production builds, OR
  - Demo controls are marked with a `data-demo` attribute that audit
    tooling can filter.
- The audit script (or a Playwright spec) confirms no demo control
  contributes to the production-control fail list.

## Notes

This is hygiene, not a user-facing bug тАФ but it directly affects audit
maintenance cost, which is itself a quality signal.
