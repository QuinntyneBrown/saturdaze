# BUG-005 — Itinerary Sunday day-switcher blanks the app with `NG04002`

**Status:** Fixed
**Severity:** High
**Opened:** 2026-05-17
**Source:** [button-link-audit-2026-05-17.md §High Priority Findings #2 + §Itinerary](../button-link-audit-2026-05-17.md)
**Area:** navigation / hrefs

## Summary

On `/itinerary`, clicking the Sunday row in the day switcher navigates to
the mock string `itinerary.html`, which the Angular router cannot match.
The result is a blank app frame and console error `NG04002`. Unlike the
bottom-nav, the day switcher's plain-click is not intercepted, so the bug
surfaces on a normal left-click.

## Steps to reproduce

1. Sign in and open <https://gentle-rock-070070c0f.7.azurestaticapps.net/itinerary>.
2. Click the Sunday entry in the day switcher.
3. Observe the URL becomes `/itinerary.html` and the SPA renders an empty
   frame.
4. Console shows `NG04002 Cannot match any routes. URL Segment: 'itinerary.html'`.

## Expected

Selecting Sunday updates the active day on `/itinerary` — either via a
real route segment (`/itinerary?day=sunday`) or in-page state — without
leaving the route.

## Actual

The plain-click follows the mock `itinerary.html` href and breaks the app.

## Likely source

- `frontend/projects/saturdaze/src/app/pages/itinerary/itinerary.page.html`

## Acceptance

- Sunday-row click switches the active day on `/itinerary` and the URL
  reflects the active day (`?day=sunday`).
- Both Saturday and Sunday rows behave the same way.
- No `NG04002` console error.
- E2E spec: open `/itinerary` → click Sunday → assert URL still starts with
  `/itinerary` and a Sunday-only assertion is visible.

## Notes

This is the same root cause as [BUG-004](BUG-004.md) — leftover mock hrefs
that the router cannot match.
