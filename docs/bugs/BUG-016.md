# BUG-016 — Splash closing "Already have one? Sign in" link has no action

**Status:** Fixed
**Severity:** Medium
**Opened:** 2026-05-17
**Source:** [button-link-audit-2026-05-17.md §Splash](../button-link-audit-2026-05-17.md)
**Area:** splash / onboarding

## Summary

The closing-section **Already have one? Sign in** link on `/` does not
navigate to `/login`. Existing users have no obvious path off the splash.

## Steps to reproduce

1. Open <https://gentle-rock-070070c0f.7.azurestaticapps.net/>.
2. Scroll to the closing CTA section.
3. Click **Already have one? Sign in**.
4. Observe: URL unchanged.

## Expected

Click navigates to `/login`.

## Actual

Nothing happens.

## Likely source

- `frontend/projects/saturdaze/src/app/pages/splash/splash.page.html`

## Acceptance

- The "Sign in" anchor is a real `[routerLink]="/login"` or click handler
  that calls `router.navigateByUrl('/login')`.
- Playwright spec: open `/` → click the closing **Sign in** link → URL is
  `/login`.
