# BUG-026 — `/weekend` detail-pane "Open full day" has no action

**Status:** Fixed
**Severity:** Medium
**Opened:** 2026-05-17
**Source:** [button-link-audit-2026-05-17.md §Home](../button-link-audit-2026-05-17.md)
**Area:** weekend / home

## Summary

The **Open full day** button inside the detail pane of `/weekend`
produces no visible change. It implies routing to a full-day view but
doesn't.

## Steps to reproduce

1. Sign in and open <https://gentle-rock-070070c0f.7.azurestaticapps.net/weekend>.
2. Expand the detail pane for a day.
3. Click **Open full day**.
4. Observe: no route, no dialog.

## Expected

Click routes to `/itinerary?day=<saturday|sunday>` showing the full
itinerary for the day in context.

## Actual

Inert button.

## Likely source

- `frontend/projects/saturdaze/src/app/pages/home/home.page.html`
- `frontend/projects/saturdaze/src/app/pages/home/home.page.ts`

## Acceptance

- Click routes to `/itinerary` with the correct day in scope.
- E2E spec covers it for both Saturday and Sunday detail panes.

The Saturday-specific variant is [BUG-028](BUG-028.md). Probably the same
underlying click handler.
