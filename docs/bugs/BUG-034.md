# BUG-034 — `/activities` "Try something new" button has no action

**Status:** Fixed
**Severity:** Medium
**Opened:** 2026-05-17
**Source:** [button-link-audit-2026-05-17.md §Activities](../button-link-audit-2026-05-17.md)
**Area:** activities

## Summary

The **Try something new** button on `/activities` produces no state
change, route, dialog, or API call. The intended behaviour isn't defined
in the deployed UI.

## Steps to reproduce

1. Sign in and open <https://gentle-rock-070070c0f.7.azurestaticapps.net/activities>.
2. Click **Try something new**.
3. Observe: nothing happens.

## Expected (decision required)

One of:

- Refresh the suggestion list with a "surprise me" call to the activity
  service.
- Open a CDK dialog that filters by family preferences (likes / dislikes /
  rhythm) and re-asks for suggestions.
- Route to a `/activities/discover` page with a guided picker.

Inert button.
