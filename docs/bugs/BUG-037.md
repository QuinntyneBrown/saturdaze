# BUG-037 — `/restaurants` "See menu" has no action

**Status:** Fixed
**Severity:** Medium
**Opened:** 2026-05-17
**Source:** [button-link-audit-2026-05-17.md §Restaurants](../button-link-audit-2026-05-17.md)
**Area:** restaurants

## Summary

The **See menu** affordance on each restaurant card produces no visible
state change. There is no link to the restaurant's menu.

## Steps to reproduce

1. Sign in and open <https://gentle-rock-070070c0f.7.azurestaticapps.net/restaurants>.
2. Click **See menu** on any card.
3. Observe: nothing — no new tab, no dialog, no route.

## Expected

Click opens the restaurant's menu URL in a new tab
(`target="_blank" rel="noopener noreferrer"`). Source the URL from the
restaurant catalogue (`Restaurant.menuUrl`).

## Actual

Inert button.

## Likely source

- `frontend/projects/saturdaze/src/app/pages/restaurants/restaurants.page.html`
- `frontend/projects/api/src/lib/models/restaurant.ts`
  (may need a `menuUrl` field)

## Acceptance

- `Restaurant` model exposes `menuUrl: string | null`.
- Cards without a `menuUrl` hide the button.
- Cards with one render an `<a>` that opens the URL in a new tab safely.
- E2E spec asserts the link target and `rel`.
