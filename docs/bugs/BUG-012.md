# BUG-012 — Splash "Get started" CTA has no action

**Status:** Fixed
**Severity:** Medium
**Opened:** 2026-05-17
**Source:** [button-link-audit-2026-05-17.md §Splash](../button-link-audit-2026-05-17.md)
**Area:** splash / onboarding

## Summary

The primary above-the-fold **Get started** button on `/` produces no
visible state change: no route, no dialog, no API call. It's the most
prominent onboarding CTA on the splash page and currently does nothing.

## Steps to reproduce

1. Open <https://gentle-rock-070070c0f.7.azurestaticapps.net/>.
2. Click **Get started**.
3. Observe: URL unchanged, page unchanged, no dialog, no network activity.

## Expected

Clicking **Get started** navigates to `/signup` so a new visitor can
create an account.

## Actual

Nothing happens.

## Likely source

- `frontend/projects/saturdaze/src/app/pages/splash/splash.page.html`
- `frontend/projects/saturdaze/src/app/pages/splash/splash.page.ts`

## Acceptance

- Click navigates to `/signup`.
- Keyboard activation (Enter / Space) works the same way.
- Playwright spec: open `/` → click **Get started** → URL is `/signup`.
