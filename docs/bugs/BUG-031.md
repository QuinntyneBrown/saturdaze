# BUG-031 — `/itinerary` footer "Regenerate" has no action

**Status:** Fixed
**Severity:** Medium
**Opened:** 2026-05-17
**Source:** [button-link-audit-2026-05-17.md §Itinerary](../button-link-audit-2026-05-17.md)
**Area:** itinerary

## Summary

The footer **Regenerate** button on `/itinerary` produces no visible
state change. Duplicate of the top-bar Regenerate ([BUG-029](BUG-029.md)).

## Steps to reproduce

1. Sign in and open <https://gentle-rock-070070c0f.7.azurestaticapps.net/itinerary>.
2. Scroll to the footer.
3. Click **Regenerate**.
4. Observe: nothing happens.

## Expected

Same behaviour as the top **Regenerate** — confirm dialog → API call →
itinerary refresh. The two buttons must share a single handler.

## Actual

Inert button.

## Likely source

- `frontend/projects/saturdaze/src/app/pages/itinerary/itinerary.page.html`

## Acceptance

- Wired to the same `regenerateDay()` method the top button uses.
- E2E spec covers both entry points reaching the same flow.
