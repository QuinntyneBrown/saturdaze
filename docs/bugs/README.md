# Bugs

Each file in this folder is one open issue, identified by a `BUG-NNN` slug. All
bugs share the same template:

```
# BUG-NNN — Title

**Status:** Open | In Progress | Fixed
**Severity:** High | Medium | Low
**Opened:** YYYY-MM-DD
**Source:** Link to the discovery doc (audit, incident, review)
**Area:** auth | nav | weekend | itinerary | ...

## Summary           — one paragraph
## Steps to reproduce — numbered
## Expected          — observable expected behaviour
## Actual            — what happens instead
## Likely source     — files most likely to need editing
## Acceptance        — concrete done-when checks
## Notes             — optional related context
```
