# BUG-023 — `/weekend` quick-action "Regenerate the weekend" uses `href="#"` and navigates to `/`

**Status:** Fixed
**Severity:** High
**Opened:** 2026-05-17
**Source:** [button-link-audit-2026-05-17.md §Home](../button-link-audit-2026-05-17.md)
**Area:** weekend / home

## Summary

The **Regenerate the weekend** quick-action card on `/weekend` is rendered
with `href="#"`. On the deployed Static Web App this anchor navigates the
user *out of the app* to `/`, breaking the session.

## Steps to reproduce

1. Sign in and open <https://gentle-rock-070070c0f.7.azurestaticapps.net/weekend>.
2. Click the **Regenerate the weekend** quick-action card.
3. Observe: URL becomes `/` (the splash). The weekend view is gone.

## Expected

Click opens a confirmation dialog ("Throw out the current plan and start
over?") and on confirm calls `POST /api/weekends/current/regenerate`
(or equivalent). The user stays on `/weekend` and sees the new plan.

## Actual

Page navigates to `/` and the user has to sign back in to their weekend
context.

## Likely source

- `frontend/projects/saturdaze/src/app/pages/home/home.page.html`
- `frontend/projects/saturdaze/src/app/pages/home/home.page.ts`

## Acceptance

- The card is a `<button>` or has `preventDefault()` on its click handler.
- Click opens a confirm dialog.
- Confirm triggers the regenerate endpoint; cancel closes the dialog.
- E2E spec covers happy path + cancel.

## Notes

This is a regression of the same shape as the bottom-nav `.html` issue
([BUG-003](BUG-003.md)) — leftover mock hrefs that the SPA can't handle.
The other two quick-action cards likely share the same `href="#"`
template — verify and roll them up here if so.
