# BUG-020 — `/weekend` "Open calendar" button has no action

**Status:** Fixed
**Severity:** Medium
**Opened:** 2026-05-17
**Source:** [button-link-audit-2026-05-17.md §Home](../button-link-audit-2026-05-17.md)
**Area:** weekend / home

## Summary

The **Open calendar** button on `/weekend` produces no visible state
change. The implied behaviour — exporting or subscribing to the weekend
plan as a calendar feed — is not implemented.

## Steps to reproduce

1. Sign in and open <https://gentle-rock-070070c0f.7.azurestaticapps.net/weekend>.
2. Click **Open calendar**.
3. Observe: no route, no dialog, no download, no API call.

## Expected (decision required)

One of:

- Download an `.ics` of the current weekend plan.
- Open a "Subscribe in calendar" dialog with `webcal://` / Google /
  iCloud / Outlook deep-links.
- Route to a `/calendar` configuration page.

Or remove the button until calendar integration exists.

## Actual

Inert button.
