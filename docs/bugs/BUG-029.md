# BUG-029 — `/itinerary` top "Regenerate" has no action

**Status:** Fixed
**Severity:** Medium
**Opened:** 2026-05-17
**Source:** [button-link-audit-2026-05-17.md §Itinerary](../button-link-audit-2026-05-17.md)
**Area:** itinerary

## Summary

The top-bar **Regenerate** button on `/itinerary` produces no visible
state change.

## Steps to reproduce

1. Sign in and open <https://gentle-rock-070070c0f.7.azurestaticapps.net/itinerary>.
2. Click the top **Regenerate** button.
3. Observe: no route, no dialog, no API call.

## Expected

Click opens a confirm dialog ("Throw out today's itinerary?") and on
confirm calls the regenerate-day endpoint scoped to the currently active
day in the day switcher.

## Actual

Inert button.

## Likely source

- `frontend/projects/saturdaze/src/app/pages/itinerary/itinerary.page.html`
- `frontend/projects/saturdaze/src/app/pages/itinerary/itinerary.page.ts`

## Acceptance

- Click → confirm dialog → API call → itinerary refresh.
- Active day is preserved across the regenerate.
- E2E spec covers happy path + cancel.
