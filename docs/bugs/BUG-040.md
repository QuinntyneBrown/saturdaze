# BUG-040 — `/saved` "More" overflow has no action

**Status:** Fixed
**Severity:** Low
**Opened:** 2026-05-17
**Source:** [button-link-audit-2026-05-17.md §Saved](../button-link-audit-2026-05-17.md)
**Area:** saved

## Summary

The **More** overflow icon on `/saved` opens no menu, route, dialog, or
API call.

## Steps to reproduce

1. Sign in and open <https://gentle-rock-070070c0f.7.azurestaticapps.net/saved>.
2. Click the **More** (overflow) icon button.
3. Observe: nothing happens.

## Expected (decision required)

Open a CDK menu of saved-list actions (Sort, Filter, Empty, Export), or
remove the button until a second top-bar action exists.

## Actual

Inert button.

## Likely source

- `frontend/projects/saturdaze/src/app/pages/saved/saved.page.html`

## Acceptance

- Either a menu opens with at least one functional action, or the button
  is removed from the template.
- E2E spec asserts the chosen behaviour.

## Notes

Sibling of [BUG-030](BUG-030.md) and [BUG-045](BUG-045.md). Consider a
shared `<sd-menu>` primitive across the three pages.
