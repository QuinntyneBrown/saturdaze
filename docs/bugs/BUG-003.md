# BUG-003 — Bottom-nav anchors expose mock `.html` hrefs

**Status:** Fixed
**Severity:** High
**Opened:** 2026-05-17
**Source:** [button-link-audit-2026-05-17.md §High Priority Findings #2](../button-link-audit-2026-05-17.md)
**Area:** navigation / hrefs

## Summary

The Angular `<sd-bottom-nav>` component still ships with the static-mock
`href` values `home.html`, `activities.html`, `saved.html`, `profile.html`.
Plain left-click is intercepted and routes correctly, but direct browser open,
copy-link, modifier-click (Ctrl/Cmd-click), and right-click → "Open in new
tab" all land on a blank app frame with console error `NG04002`.

## Steps to reproduce

1. Open <https://gentle-rock-070070c0f.7.azurestaticapps.net/profile>.
2. Right-click any bottom-nav item (e.g. "Home") → "Copy Link Address".
3. Paste it into a new tab and load it directly.
4. Observe the app frame is blank; the DevTools console shows
   `NG04002 Cannot match any routes. URL Segment: 'home.html'`.
5. Also reproducible by Ctrl/Cmd-clicking any bottom-nav item.

## Expected

The four bottom-nav anchors point to real Angular routes:

| Label      | href         |
|------------|--------------|
| Home       | `/weekend`   |
| Activities | `/activities`|
| Saved      | `/saved`     |
| Profile    | `/profile`   |

Direct opens, copy-link, modifier-click, and "Open in new tab" all load the
correct route.

## Actual

`<a href="home.html">` (etc.) is rendered, which the Angular router cannot
match. Only the click handler's `event.preventDefault()` masks the bug for
plain left-click.

## Likely source

- `frontend/projects/components/src/lib/bottom-nav/bottom-nav.ts`
- `frontend/projects/components/src/lib/bottom-nav/bottom-nav.html`

## Acceptance

- The `items` map inside `bottom-nav.ts` uses real route paths in `href`.
- Direct open of `/weekend`, `/activities`, `/saved`, `/profile` works.
- Ctrl/Cmd-click on any bottom-nav item opens that route in a new tab.
- No `NG04002` console error during navigation.
- E2E selectors prefer `data-nav-key` over `href$="home.html"` (the audit
  notes this; the sign-out e2e already uses `data-nav-key`).

## Notes

If mock parity is a hard requirement, add Angular redirect routes for
`home.html → /weekend`, `activities.html → /activities`, `saved.html → /saved`,
`profile.html → /profile`. Otherwise just fix the hrefs.
