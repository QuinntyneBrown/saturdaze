# BUG-004 — Home day card href is the mock `itinerary.html`

**Status:** Fixed
**Severity:** High
**Opened:** 2026-05-17
**Source:** [button-link-audit-2026-05-17.md §High Priority Findings #2](../button-link-audit-2026-05-17.md)
**Area:** navigation / hrefs

## Summary

The day cards on `/weekend` (Home) render an anchor whose `href` is the
mock string `itinerary.html`. The plain-click handler intercepts and routes
to `/itinerary`, but modifier-click, copy-link, and direct open of
`itinerary.html` blank the app with `NG04002`.

## Steps to reproduce

1. Open <https://gentle-rock-070070c0f.7.azurestaticapps.net/weekend>.
2. Right-click a day card → "Copy Link Address".
3. Paste into a new tab and load it.
4. The app frame is blank with console error
   `NG04002 Cannot match any routes. URL Segment: 'itinerary.html'`.
5. Ctrl/Cmd-clicking the card also opens an `itinerary.html` URL that
   produces the same failure.

## Expected

The card anchor points to `/itinerary` (optionally with a `day` query) and
behaves identically for plain-click, modifier-click, direct open, and
copy-link.

## Actual

`href="itinerary.html"` is rendered. Only the JS click handler hides the
bug for default left-click.

## Likely source

- `frontend/projects/components/src/lib/day-card/day-card.ts`
- `frontend/projects/saturdaze/src/app/pages/home/home.page.html`

## Acceptance

- Day-card `href` is a real Angular route (`/itinerary` or
  `/itinerary?day=saturday`).
- Ctrl/Cmd-click opens that route in a new tab.
- Direct open of the same URL works.

## Notes

Pair with [BUG-005](BUG-005.md) — the itinerary day switcher has the same
class of failure.
