# BUG-030 — `/itinerary` top-bar "More" has no action

**Status:** Fixed
**Severity:** Low
**Opened:** 2026-05-17
**Source:** [button-link-audit-2026-05-17.md §Itinerary](../button-link-audit-2026-05-17.md)
**Area:** itinerary

## Summary

The top-bar **More** overflow button on `/itinerary` produces no menu,
route, dialog, or visible state change.

## Steps to reproduce

1. Sign in and open <https://gentle-rock-070070c0f.7.azurestaticapps.net/itinerary>.
2. Click the **More** (overflow) icon button.
3. Observe: nothing happens.

## Expected (decision required)

One of:

- Open a CDK menu / action sheet with itinerary actions (Share, Export,
  Reset, etc.).
- Remove the button until a second top-bar action exists.

## Actual

Inert button.

## Likely source

- `frontend/projects/saturdaze/src/app/pages/itinerary/itinerary.page.html`

## Acceptance

- Either a menu opens with at least one functional action, or the button
  is removed from the template.
- E2E spec asserts the chosen behaviour.

## Notes

Same pattern as the `/profile` more button ([BUG-045](BUG-045.md)). If both
end up needing a menu primitive, build one shared `<sd-menu>` rather than
ad-hoc CDK overlay attachments.
