# BUG-018 — Signup "Privacy Policy" link uses `href="#"`

**Status:** Fixed
**Severity:** Low
**Opened:** 2026-05-17
**Source:** [button-link-audit-2026-05-17.md §Signup](../button-link-audit-2026-05-17.md)
**Area:** legal / nav

## Summary

The **Privacy Policy** link in the legal-acceptance copy beneath the
signup form is rendered with `href="#"`. Clicking it scrolls to top
rather than opening a privacy document.

## Steps to reproduce

1. Open <https://gentle-rock-070070c0f.7.azurestaticapps.net/signup>.
2. Click the inline **Privacy Policy** link.
3. Observe: URL becomes `/signup#`, scroll jumps to top.

## Expected

The Privacy link opens the actual Privacy Policy document — via internal
route, external URL, or static page.

## Actual

`href="#"` traps the user on `/signup`.

## Likely source

- `frontend/projects/saturdaze/src/app/pages/signup/signup.page.html`

## Acceptance

- Same options as [BUG-017](BUG-017.md): route to `/privacy`, external URL
  with `target="_blank" rel="noopener"`, or plain text.
- Playwright spec asserts the chosen behaviour.
