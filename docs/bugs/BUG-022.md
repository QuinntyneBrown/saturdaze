# BUG-022 — `/weekend` hero "Plan This Weekend" button has no action

**Status:** Fixed
**Severity:** Medium
**Opened:** 2026-05-17
**Source:** [button-link-audit-2026-05-17.md §Home](../button-link-audit-2026-05-17.md)
**Area:** weekend / home

## Summary

The hero primary CTA **Plan This Weekend** on `/weekend` produces no
visible state change. It's the most prominent action on the page and is
inert.

## Steps to reproduce

1. Sign in and open <https://gentle-rock-070070c0f.7.azurestaticapps.net/weekend>.
2. Click **Plan This Weekend**.
3. Observe: nothing — no route, no API call, no state change.

## Expected

Click triggers a generate-weekend flow:

- If there is no current weekend plan: call the generate endpoint, show
  loading state, and reveal the plan when the response lands.
- If there is one already: route to `/itinerary` or open a regenerate
  confirm dialog (see [BUG-027](BUG-027.md)).

## Actual

Inert button.

- `frontend/projects/saturdaze/src/app/pages/home/home.page.html`
- `frontend/projects/saturdaze/src/app/pages/home/home.page.ts`

  (likely already has a generate method that needs wiring)

## Acceptance

- Click triggers the generate-weekend flow with a loading state.
- After success, the home view reflects the new plan.
- E2E spec covers the click path.
