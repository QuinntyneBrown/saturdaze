# BUG-035 — `/restaurants` "Refresh picks" has no action

**Status:** Fixed
**Severity:** Medium
**Opened:** 2026-05-17
**Source:** [button-link-audit-2026-05-17.md §Restaurants](../button-link-audit-2026-05-17.md)
**Area:** restaurants

## Summary

The **Refresh picks** button on `/restaurants` produces no visible state
change. Re-rolling the suggested restaurants is a core product loop and
is not wired.

## Steps to reproduce

1. Sign in and open <https://gentle-rock-070070c0f.7.azurestaticapps.net/restaurants>.
2. Click **Refresh picks**.
3. Observe: card list unchanged, no API call, no loading state.

## Expected

Click calls the restaurant service (`POST /api/restaurants/suggestions` or
similar) and replaces the current cards with a new shuffle, while
preserving any locked / voted picks.

## Actual

Inert button.

## Likely source

- `frontend/projects/saturdaze/src/app/pages/restaurants/restaurants.page.html`
- `frontend/projects/saturdaze/src/app/pages/restaurants/restaurants.page.ts`

## Acceptance

- Click triggers a refresh with a loading state.
- New card set rendered on success.
- E2E spec covers it.
