# BUG-043 — `/errand` "Pick a different slot" has no action

**Status:** Fixed
**Severity:** Medium
**Opened:** 2026-05-17
**Source:** [button-link-audit-2026-05-17.md §Errand](../button-link-audit-2026-05-17.md)
**Area:** errand

## Summary

The **Pick a different slot** button on `/errand` produces no visible
state change. The errand-scheduling slot picker is unimplemented.

## Steps to reproduce

1. Sign in and open <https://gentle-rock-070070c0f.7.azurestaticapps.net/errand>.
2. Click **Pick a different slot**.
3. Observe: nothing — no dialog, no time-slot list, no API call.

## Expected

Click triggers the slot-finder logic for the errand (given its duration
and location) and renders alternate time-slot choices the family can
adopt. The choices are derived from the current weekend's free time
between commitments / blocks.

## Actual

Inert button.

## Likely source

- `frontend/projects/saturdaze/src/app/pages/errand/errand.page.html`
- `frontend/projects/saturdaze/src/app/pages/errand/errand.page.ts`
- Backend: an errand / slot-finder application service (likely missing).

## Acceptance

- Click reveals alternate slot choices (chips, list, or dialog).
- Choosing one updates the form state for [BUG-044](BUG-044.md)'s submit.
- E2E spec covers the slot-selection flow.

## Notes

The errand form inputs themselves are not bound to state today — see the
audit note "Bind the form inputs to state."
