# BUG-033 — `/itinerary` detail "See on map" has no action

**Status:** Fixed
**Severity:** Medium
**Opened:** 2026-05-17
**Source:** [button-link-audit-2026-05-17.md §Itinerary](../button-link-audit-2026-05-17.md)
**Area:** itinerary

## Summary

The **See on map** affordance inside each itinerary detail block produces
no visible state change. There is no map UX in the deployed app.

## Steps to reproduce

1. Sign in and open <https://gentle-rock-070070c0f.7.azurestaticapps.net/itinerary>.
2. Expand a block in the detail view.
3. Click **See on map**.
4. Observe: nothing — no route, no dialog, no native map open.

## Expected

One of (decision required):

- Route to a `/map?day=<day>` page rendering all block locations on a
  map embed (Google Maps / Mapbox / OpenStreetMap iframe).
- Open a CDK dialog with the same map embed.
- Deep-link to Google Maps via
  `https://www.google.com/maps/dir/?api=1&...` with the day's stops as
  waypoints (`target="_blank"`).

## Actual

Inert affordance.
