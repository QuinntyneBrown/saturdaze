# Support

Use GitHub Issues for reproducible bugs, feature requests, and documentation
problems.

## Bugs

Please include:

- a short summary of the problem;
- reproduction steps;
- expected behavior;
- actual behavior;
- route, command, or workflow involved;
- screenshots, logs, terminal output, or Playwright traces when relevant;
- operating system, browser, .NET SDK version, Node.js version, and npm version.

## Feature Requests

Please include:

- the user problem the feature solves;
- the workflow or page it affects;
- any design constraints or examples;
- whether it changes backend APIs, database schema, frontend behavior, or tests.

## Questions

For setup or usage questions, include the command you ran and the output you
received. If the question is about the app behavior, include the route and any
seed data or profile state needed to reproduce it.

## Security

Do not use public issues for vulnerabilities. Follow the process in
`SECURITY.md`.
